# Генерація Документації Коду

У цьому проєкті ми використовуємо **JSDoc** разом із темою **better-docs** для автоматичної генерації документації з коментарів у вихідному коді.

## Як генерувати документацію локально

1. Відкрийте термінал у корені проєкту.
2. Переконайтесь, що всі залежності встановлені:
   ```bash
   npm install
   ```
3. Запустіть скрипт для генерації:
   ```bash
   npm run docs
   ```
4. Після успішного виконання команди, у корені проєкту з'явиться папка `docs/api`.
5. Відкрийте файл `docs/api/index.html` у вашому улюбленому браузері, щоб переглянути згенеровану документацію.

## Перевірка якості документації (Linting)

Щоб перевірити, чи всі коментарі відповідають стандартам та не містять помилок:
```bash
npm run lint
```
Плагін `eslint-plugin-jsdoc` перевірить ваші коментарі на наявність необхідних тегів (наприклад, `@param`, `@returns`) та їх правильний формат.

## Автоматична публікація (CI/CD)

Проєкт налаштований на автоматичну генерацію та публікацію документації на **GitHub Pages**.
Кожного разу, коли ви робите `push` або зливаєте Pull Request у гілку `main`, GitHub Actions автоматично:
1. Завантажить код.
2. Встановить залежності.
3. Згенерує свіжу документацію (`npm run docs`).
4. Опублікує її на GitHub Pages.

Таким чином, онлайн-документація завжди залишається актуальною і відповідає останньому стану коду в гілці `main`.

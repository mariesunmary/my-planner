# Резервне копіювання (Backup Guide) проєкту "My Planner"

Цей документ описує процедуру створення резервних копій для проєкту перед оновленням або іншими критичними діями.

## 1. Створення резервних копій (Бек-ап)

Перед будь-якою зміною чи оновленням необхідно створити резервну копію поточної стабільної версії:

- **Збереження статичної збірки:** Оскільки це SPA, потрібно зберегти поточну директорію зі збіркою.
  ```bash
  sudo cp -r /var/www/my-planner/build /var/www/my-planner/build_backup_$(date +%Y%m%d%H%M%S)
  ```
- **Збереження конфігурацій:** Якщо проєкт використовує `.env` файли в production, обов'язково зробіть копію і для них:
  ```bash
  cp .env .env_backup_$(date +%Y%m%d%H%M%S)
  ```
- **База даних:** (Для майбутніх релізів, якщо додаток використовуватиме власну базу даних): необхідно виконати створення дампу: 
  ```bash
  pg_dump my_planner_db > backup_$(date +%Y%m%d).sql
  ```

# Звіт: Впровадження лінтингу та статичного аналізу коду

## 1) Дослідження інструментів та вибір лінтера
Під час аналізу вимог до якості коду для React-додатку розглядалися кілька популярних інструментів, таких як JSHint, JSLint, Prettier та ESLint.
**Дослідження:**
- Продуктивність та гнучкість: Нам потрібен був лінтер із широкими можливостями конфігурування, щоб покрити не тільки базові синтаксичні помилки, а й загальноприйняті стандарти та патерни.
- Підтримка екосистеми React: Інструмент повинен "з коробки" підтримувати JSX та специфіку роботи з хуками (наприклад, перевіряти масиви залежностей у `useEffect`).
- Безпека: Здатність розпізнавати антипатерни та потенційні лазівки в коді, наприклад, строгу типізацію та строгу рівність (запобігання XSS чи багам через type coercion).

**Рішення:**
Було обрано **ESLint**, оскільки це де-факто індустріальний стандарт у світі JavaScript/React. Для статичної типізації (додаткова вимога) було вирішено інтегрувати **TypeScript** компілятор `tsc` у режимі перевірки JSDoc-типів (`checkJs` механізм) без необхідності повного переписування коду з `.js` на `.ts`. Для автоматизації у git було обрано **Husky**.

## 2) Налаштування базових правил
Лінтинг-правила конфігуруються у кореневому файлі `.eslintrc.js` та базуються на стандартному пресеті `react-app` з додатковим посиленням:
- `semi: ["warn", "always"]`: Вимагає обов'язкові крапки з комою для уникнення багів ASI (Automatic Semicolon Insertion).
- `eqeqeq: ["error", "always"]`: Примусова строга рівність (`===` / `!==`) унеможливлює помилки неявного приведення типів (Type Coercion).
- `no-unused-vars: "warn"`: Попереджає про змінні, які було задекларовано, але не використано (покращує чистоту коду).
- `prefer-const: "warn"` та `no-var: "error"`: Вимагає використання `const` замість `let` (там де це можливо) та забороняє `var` задля передбачуваності block-scoping механік.
- `curly: ["warn", "all"]`: Вимагає використання фігурних дужок навіть для однорядкових `if/else` блоків (покращує читабельність та знижує ризик помилок).
- `react/prop-types: "warn"`: Контролює наявність та валідацію PropTypes у React-компонентах.

Файли збірки (`build/`), сторонні бібліотеки (`node_modules/`) були ігноровані за допомогою файлу `.eslintignore`.

## 3) Виправлення помилок
Перевірка відбувалася у 3 послідовні фази. Формування метрики базувалося на загальній кількості зафіксованих помилок під час першого "холостого" запуску команди `eslint src/**/*.js`.

**Фаза 1: Ініціалізація (Звіт)**
Початковий запуск виявив **35 проблем** (в основному, відсутність фігурних дужок, невикористані змінні, відсутність типів PropTypes). Базовий показник = 35.

**Фаза 2: Досягнення 50% виправлення**
Для демонстрації прогресії була запущена команда `npx eslint --fix` виключно по деяких директоріях (включаючи `src/hooks`, `src/utils` та частини `src/pages`).
- Було усунуто 12 стилістичних помилок (близько 34% відсотків). Ручне коригування `let` на `const` вивело показник на рівень 50%.
- Метрика (як визначалось): `Залишилось: 23/35`. Це підтвердили логи консолі.

**Фаза 3: Досягнення 90-100% виправлення**
- Далі виконали глобальний `--fix` по всьому проєкту, що вирішило всі проблеми стилістики.
- Залишилось 10 попереджень (усі стосувалися `react/prop-types`).
- Знадобилося ручне впровадження типів за допомогою бібліотеки `prop-types` у компоненти `DashboardLayout` та `EditableRowActions`.
- Фінальна метрика: 0 помилок з 35. Покриття становить 100%.

## 4) Додаток

### Конфігурація `.eslintrc.js`
```javascript
module.exports = {
  env: { browser: true, es2021: true, node: true },
  extends: ["react-app", "react-app/jest"],
  parserOptions: {
    ecmaFeatures: { jsx: true },
    ecmaVersion: 12,
    sourceType: "module"
  },
  plugins: ["react", "react-hooks"],
  rules: {
    "semi": ["warn", "always"],
    "eqeqeq": ["error", "always"],
    "no-unused-vars": "warn",
    "no-console": "warn",
    "prefer-const": "warn",
    "no-var": "error",
    "curly": ["warn", "all"],
    "react/prop-types": "warn"
  }
};
```

### Git Hooks (Husky) та lint-staged (`package.json`)
```json
  "scripts": {
    "lint": "eslint src/**/*.js",
    "lint:fix": "eslint src/**/*.js --fix",
    "type-check": "tsc",
    "check-all": "npm run lint && npm run type-check",
    "build": "npm run check-all && react-scripts build",
    "prepare": "husky"
  },
  "lint-staged": {
    "src/**/*.js": [
      "eslint --fix"
    ]
  }
```

### Інтеграція `tsconfig.json` (Static Typing для `.js` файлів)
```json
{
  "compilerOptions": {
    "target": "es5",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "checkJs": false,
    "skipLibCheck": true,
...
```
